# 1. 目的

「共有設定」タブにおける削除操作の誤解を防ぐため、エディター管理画面と同様の「ソフトデリート（削除マーク）」UIを実装する。また、適用ボタンの視認性を向上させ、未適用のままアプリを閉じた場合のデータ喪失を防ぐ。

## 提案された解決策
1. `ShareCard` クラスの削除動作を変更し、行を消さずに入力欄をグレーアウトさせ、「元に戻す」ボタンを提供する。
2. アプリ全体の適用ボタンに `sv_ttk` の `Accent.TButton` スタイルを適用し、プライマリアクションとして目立たせる。
3. アプリケーション終了時（`app.py`）に未適用の変更を検知し、警告ダイアログを表示する。

# 2. 変更内容

## 2.1 `ShareCard` のUI改善
- `_confirm_delete` を廃止し、`_toggle_delete` を実装。
- 削除時は `_set_state` によりすべての入力フィールド（Entry, Combobox, Checkbutton 等）を `tk.DISABLED` にする。
- 警告ラベル「⚠ 「適用」をクリックすると削除されます」を表示する。

## 2.2 `SmbConfEditorApp` の改善
- 適用ボタンを `ttk.Button(..., style="Accent.TButton")` に変更。
- ウィンドウの `WM_DELETE_WINDOW` プロトコルをフックし、`_on_closing` メソッドで未保存の変更（削除待ちの共有フォルダ、または入力中の新規共有フォルダ）が存在するかチェックする機能を実装。
